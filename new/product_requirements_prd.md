# 积分联盟计划iOS版App - 产品需求文档 (PRD)

## 项目概述

### 1.1 项目名称
积分联盟计划iOS版App，一款赋能用户将各种渠道积分进行兑换、商城消费的基础应用

### 1.2 项目目标
- 整合多种积分渠道（支付宝、和积分、苹果积分、抖音积分等）
- 提供具备质量的积分兑换和商城消费体验
- 实现“积分联盟+积分查询+积分商城”的完整生态

### 1.3 目标用户
- 对积分转值有需求的经典用户
- 希望通过积分证贵拥有更多优惠和权益的用户
- 商家及政府联盟参与方

### 1.4 项目范围
- 创建一款可用于iOS设备的积分联盟应用
- 支持消费者在各种场景下使用积分
- 实现消费者与商家的实时交互
- 提供完整的积分解决方案

---

## 需求分析

### 2.1 用户需求

**基础需求**
- 显示应有的积分余额
- 查询各种货币的换取率
- 实时查看积分获取记录
- 支持多种兑换方式

**加速需求**
- 10s内完成核销操作
- 400ms内响应用操作
- 95%+ 的操作成功率

### 2.2 商业模式
- 免费推广+特惠支持
- 投入式增长（新用户获取）
- 商家发布+用户活跃

## 功能要求

### 3.1 核心功能模块

#### 3.1.1 平台积分余额模块
**功能描述**：
- 展示当前积分余额（具体数值）
- 可见性设置：数字显示/图形化
- 实时更新，即时反馈

**技术要求**
- 响应式布局，适配iPhone各种屏幕
- 支持Force Touch功能
- 无障碍访问

#### 3.1.2 人民币法币价值模块
**功能描述**：
- 显示币种和中心货币
- 实时换算支持
- 法币类型支持(外币/区块链)
- 重点显示实时价值

**技术要求**
- 实时换算算法
- 支持跨平台换算
- 更新换算收费实时显示

#### 3.1.3 积分渠道兑入兑出列表
**功能描述**：
- 支持多种软件源积分
- 积分混合兑换（分次步进）
- 支持即时兑换
- 完整的交易记录

**技术要求**
- 实时更新渠道信息
- 支持点击兑换链接
- 后台管理在线创建

#### 3.1.4 积分商城模块
**功能描述**：
- 商品分类（服务/商品/套餐）
- 搜索功能
- 商品详情展示
- 支持积分+现金混合支付
- 热门推荐
- 限时活动支持

**技术要求**
- 商品库管理后台
- 网页API插件
- 多维度筛选算法
- 订单管理系统融合

#### 3.1.5 积分订单模块
**功能描述**：
- 订单创建与管理
- 订单状态跟踪（未支付/支付中/已完成）
- 交易记录完整保存
- 物流信息显示
- 售后支持（退货/改单）

**技术要求**
- 完整的命周周期管理
- 支持历史记录查询
- 实时跟踪状态变化
- 售后支持通过API和邮箱通知

#### 3.1.6 记录与查看模块
**功能描述**：
- 兑换历史记录
- 搜索条件过滤
- 时间范围筛选
- 记录细度
- 对格列表

**技术要求**
- 数据过滤/排序
- 切换视图
- 切换页码

### 3.2 附加功能模块

#### 3.2.1 准备金功能
- 自动累积到阈值后立即兑换
- 提供兑换预约
- 支持分期自动累积

#### 3.2.2 文件上传
- 关键词搜索
- 分类筛选
- 时间排序

#### 3.2.3 数据统计
- 积分用量趋势图
- 收益指标
- 淘汰率

## 视觉设计

### 4.1 界面布局

#### 4.1.1 布局原则
- Figma设计框架
- 4则坐标系：左上（常见）、右上（快捷路由）、下方（常见）、单页（深度）
- 响应式设计，适配iPhone各屏大小

#### 4.1.2 页面区域划分
- 首页：积分概览 + 快速兑换
- 积分页：显示详细信息
- 商城页：商品流水布
- 订单页：交易记录
- “我的”页：个人中心

### 4.2 颜色方案

#### 4.2.1 主题色
- 主色调：支付宝蓝(#1677FF)
- 辅助色：绿色(#10B981)
- 背景色：白色/浅灰色

#### 4.2.2 特殊组件
- 积分卡片：圆角设计，边框渐变
- 进度条：环形设计，表示兑换进度
- 按钮：圆角设计，强调色

### 4.3 交互设计

#### 4.3.1 操作反馈
- 点击效果：轻微放大放缩
- 加载状态：显示转动阶段
- 成功反馈：绿色标识符
- 错误处理：清晰的错误提示

#### 4.3.2 操作流程
- 兑换流程：选清单 → 确认 → 支付 → 完成
- 订单流程：创建 → 支付 → 确认 → 查看
- 所有页面支持手势操作

## 产品路由

### 5.1 版本发布
- **v1.0.0**：核心功能实现
- **v1.1.0**：优化用户体验
- **v2.0.0**：商业模式全面升级

### 5.2 技术栈
- 前端：SwiftUI + Combine
- 后端：RESTful API
- 数据库：Core Data / CloudKit
- 第三方API：支付宝/和积分交易API

### 5.3 风险和应对
- 风险：用户数据安全、支付安全
- 应对：强化密码加密、安全认证
